Resum

La funció principal d’un servidor d’aplicacions web és enviar pàgines web en resposta a les peticions dels clients. Una pàgina web està formada per codi HTML (en anglès HyperText Markup Language, llenguatge de marques d’hypertext) i qualsevol contingut addicional que es pot incloure en un document, com ara imatges, fulls d’estil i scripts.

Un client, normalment un navegador web o un robot web, inicia la comunicació mitjançant una sol·licitud d’un recurs específic a través d’HTTP i el servidor respon amb el contingut d’aquest recurs o un missatge d’error si no pot fer-ho.

El servidor HTTP Apache és un servidor web HTTP de codi obert, per a moltes plataformes: Unix, Microsoft Windows, Macintosh i d’altres. És el servidor web més utilitzat en sistemes Linux.

Apache es configura posant directives en arxius de configuració de text pla. L’arxiu de configuració es diu apache2.conf i per defecte es troba a la ruta /etc/apache2/apache2.conf. Els canvis en els arxius de configuració principals són només reconeguts per Apache quan s’inicia o es reinicia.

Apache és un servidor modular. Això implica que només la funcionalitat més bàsica s’inclou en el servidor central. Les característiques esteses estan disponibles a través de mòduls que es poden carregar a Apache. Un dels principals motius pels quals s’utilitzen els mòduls amb Apache és que no totes les instal·lacions de servidors web necessiten les mateixes funcionalitats.

MySQL és un sistema de gestió de bases de dades relacional (en anglès RDBMS - Relational DataBase Management System), multifil i multiusuari, que usa el llenguatge SQL (Structured Query Language). Molts projectes de programari lliure que requereixen un sistema de base de dades de gestió utilitzen MySQL. Entre els més famosos trobem: TYPO3, Joomla, WordPress, phpBB, Drupal… MySQL també s’utilitza en molts llocs de grans dimensions com ara Wikipedia, Facebook, i Twitter.

El codi resultant dels llenguatges script de client és el que s’executa en el navegador web (Firefox, IE, Chrome, Opera, Safari) de l’usuari i normalment forma part del codi HTML d’un document. Els llenguatges script de client van ser dissenyats per afegir interactivitat a les pàgines HTML. Són llenguatges de programació lleugers i interpretats (és a dir, que els scripts s’executen sense una compilació preliminar).

L’avantatge principal del llenguatge d’script de client és que no satura la CPU del servidor i no consumeix amplada de banda. S’utilitza principalment per a tasques que no requereixen comunicació amb el servidor. JavaScript és el llenguatge de referència per programar codi en el cantó del client.

Les aplicacions web desenvolupades amb llenguatges script de servidor s’executen directament en el servidor web i produeixen com a resultat codi HTML dinàmic en què el client no té accés al codi font del servidor. La finalitat principal és proporcionar molta interacció amb l’usuari segons els seus requeriments o criteris. PHP és un dels llenguatges més utilitzats per a programar codi en el cantó del servidor.

Instal·lar i configurar un servidor Apache, un servidor MySQL i el llenguatge PHP, així com configurar-ho perquè s’interrelacionin i el servidor funcioni perfectament és una tasca complexa que només poden emprendre informàtics professionals.

Per simplificar la tasca d’instal·lar Apache + PHP + MySQL en Linux, Windows o Mac OS i apropar al gran públic la possibilitat de gaudir d’aquests serveis, hi ha els anomenats paquets LAMP, WAMP o MAMP que instal·len i configuren automàticament aquestes aplicacions per a Linux, Windows o Mac OS.

Durant el procés d’instal·lació i configuració d’un servidor d’aplicacions web cal prendre nota del tots els passos que s’han seguit. En primer lloc en el moment de la instal·lació i després en el moment de la configuració, de manera que es tingui una referència completa de tot el que s’ha fet. Conèixer els detalls de per què s’ha posat un valor a una variable a un fitxer de configuració pot ser molt útil en el futur, ja que és probable que passat un temps us n’hagueu oblidat.

La documentació també és importat per a la transmissió de coneixement, podeu passar-la a altres persones que poden aprendre com està muntat el servidor sense haver d’explicar-ne res.

El llenguatge PHP és uns dels llenguatges de guions de servidor més utilitzat. El PHP va ser creat l’any 1994 per Rasmus Lerdorf, que va crear un seguit d’utilitats i macros, les va anomenar Personal Home Page Tools (PHP) i les va posar a la lliure disposició de tota la comunitat d’Internet.

El PHP va passar a ser un llenguatge de programació molt potent, de codi obert i acceptat àmpliament en entorns comercials. És una eina viable per a la programació en entorns empresarials.

Una de les característiques de PHP és que és extremadament simple per al principiant, però ofereix moltes característiques avançades per als programadors professionals.

La sintaxi de PHP és molt semblant a la de C o C++. El més destacat passa a nivell semàntic: no és un llenguatge tipat (no es fa comprovació de tipus), és a dir, quan creem una variable podem emmagatzemar en ella el tipus de dades que vulguem. Això és molt flexible i còmode per al desenvolupador, encara que els errors que es cometen poden ser molt més greus i difícils de corregir. Les possibilitats de l’intèrpret per detectar incompatibilitats entre variables es redueixen molt i poden passar coses estranyes en determinades circumstàncies si no es té cura.

Per editar en codi PHP només és necessari un editor simple de text pla. El que s’aconsegueixi ja depèn de les habilitats i la creativitat del programador. No obstant això, per fer la vida del programador una mica més fàcil, existeixen alguns editors de codi font amb funcions d’edició avançades. Aquestes funcions les podem trobar durant la codificació, la depuració i en el moment de fer les proves. Entre els editors de codi obert que funcionen a moltes plataformes per exemple trobem Netbeans i Eclipse PHP Developer Tools (PDT).

A PHP l’usuari pot definir totes les funcions que necessiti. És imprescindible que el nom donat a la funció no estigui prohibit, és a dir, no es poden fer servir paraules reservades (per exemple, echo), començar amb un número o un caràcter especial ni repetir el nom d’una funció dins un mateix àmbit. També té funcions definides internament. Aquestes funcions són de gran utilitat per al programador, com per exemple echo i print.

El formulari electrònic és una eina molt potent, fortament establida a Internet. Permet l’intercanvi o la introducció de dades entre usuaris d’una manera ràpida i senzilla, quan HTML i PHP treballen amb formularis ens trobem amb un binomi gairebé ideal.

Les sessions a un lloc web són una eina que permet emmagatzemar informació específica de l’usuari visitant en variables. El llenguatge del PHP inclou una sèrie de funcions molt intuïtives i senzilles d’utilitzar, que no solament permetran crear sessions, sinó que us donaran gran llibertat per treballar amb dades referents a l’usuari.

El PHP és el llenguatge d’script de referència però no és l’únic. Amb la tecnologia Microsoft existeixen les pàgines ASP (en anglès Active Server Pages, pàgines actives de servidor), i les seves succesores ASP.NET que fan servir el marc de programació NET. També en el món de java, existeixen les pàgines JSP (en anglès JavaServer Pages, Pàgines de java de servidor), que necessiten un servidor web Apache Tomcat per funcionar.

L’accés a bases de dades és una de les accions més habituals que es realitzen en els llenguatges d’script de servidor. Afegir una base de dades a un lloc web proporciona els mitjans per crear contingut dinàmic i facilita la interactivitat amb l’usuari. Per exemple el llenguatge d’script de servidor PHP permet interactuar amb la gran majoria de bases de dades existents, com ara MySQL, PostgreSQL, Oracle, etc.

Una base de dades és un conjunt de dades que s’emmagatzemen independentment de la manera amb què es volen recuperar. Hi ha molts sistemes de bases de dades però MySQL és un dels més utilitzats, ja que és gratuït i està disponible per la majoria de sistemes Unix/Linux, així com per Windows.

Amb una base de dades es poden fer diferents operacions: consultar dades, inserir dades, actualitzar dades o esborrar dades. Per a totes aquestes operacions l’estructura del codi de programació és exactament igual. Sempre que es programa una connexió a una base de dades MySQL s’han de fer uns passos determinats.

En primer lloc, fer una connexió al servidor MySQL. Per fer aquesta operació s’utilitza la funció mysql_connect(). La funció retorna un valor del tipus resource (recurs), que representa la connexió amb la base de dades.

En segon lloc, seleccinar la base de dades amb la qual es vol treballar amb la funció mysql_select_db(). S’ha d’indicar la connexió amb el servidor de base de dades i la base de dades amb la qual es vol treballar.

En tercer lloc, s’ha de fer la consulta/operació necessària amb la funció mysql_query() i si és una operació de consulta de dades, llegir les dades que retorna la consulta SQL.

Per verificar el número de registres que retorna una consulta es pot fer servir la sentència mysql_affected_rows() que pot ser cridada immediatament després de l’execució d’una sentència amb mysql_query().

La funció mysql_error() retorna la descripció de l’error de l’última operació que s’ha realitzat a MySQL. Aquesta funció retorna una cadena buida (””) si no hi ha error. Els errors que vénen de la base de dades no emeten avisos. Al seu torn, utilitzen la funció mysql_errno() per recuperar text amb errors. Aquesta funció només retornarà el text amb errors de la funció més recent executada per MySQL (sense incloure mysql_error() i mysql_errno()). És a dir, si es vol utilitzar cal assegurar-se de revisar el valor abans de cridar una altra funció de MySQL.

Per a totes les connexions, consultes, i altres operacions que els usuaris poden intentar realitzar, MySQL utilitza seguretat basada en llistes de control d’accés (ACLs). També hi ha algun suport per a connexions xifrades mitjançant SSL entre clients i servidors MySQL.

Per verificar el funcionament de MySQL es pot fer servir programari de monitorització que controla la disponibilitat de serveis. Per exemple, entre les eines de programari de monitorització lliures més importants tenim Nagios i Pandora FMS.
