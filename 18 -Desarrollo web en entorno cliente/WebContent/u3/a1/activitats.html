Activitats

Ús d'objectes

L’objectiu d’aquesta activitat és practicar la creació d’objectes de forma literal, actualitzar-los, augmentar-los i eliminar-ne propietats i mètodes.

L’empresa Entreteniments IOC, SL està treballant en un joc de zombis i us ha demanat que genereu una estructura de dades per representar els jugadors. Com que es tracta d’una versió alfa no cal que us preocupeu per l’encapsulament de la informació, els interessa que sigui el més simple possible.

L’estructura de dades ha de tenir les propietats següents:

  • Nom
  • Cognom
  • Ocupació
  • Nivell
  • Facció

També us demanen que creeu una llista (array) de jugadors amb dos jugadors d’exemple, fent servir aquesta estructura.

Per facilitar la comprovació de les dades dels jugadors ús demanen que afegiu un mètode toString() a l’estructura de dades per poder veure la informació en una sola línia i demostreu que funciona correctament.

Finalment, es considera que un jugador podrà abandonar la seva facció, així que us demanen que elimineu la facció del segon jugador i en demostreu el funcionament correcte.

Com que es demana expressament la implementació més simple possible i no cal preocupar-se per l’encapsulament la millor opció és fer servir un objecte declarat literalment: 

  1. let jugador = {
  2.   nom: 'Nom del jugador',
  3.   cognom: 'Cognom del jugador',
  4.   ocupacio: 'Sense ocupació',
  5.   nivell: 0,
  6.   faccio: 'Sense facció',
  7. }

A continuació s’ha de crear un array de jugadors i seguidament creem dues còpies d’aquest objecte amb Object.create(): 

  1. let jugadors = [];
  2. jugadors.push(Object.create(jugador));
  3. jugadors.push(Object.create(jugador));

Podeu afegir la línia console.log(jugadors); per comprovar que s’han afegit dues instàncies de jugador. Com que us demanen que siguin dos jugadors d’exemple, les seves dades han de ser diferents, així doncs s’han d’actualitzar. Podeu aprofitar l’índex per obtenir la referència de l’objecte dins de l’array: 

  1. jugadors[0].nom = 'Ricard';
  2. jugadors[0].cognom = 'Ensutjs';
  3. jugadors[0].ocupacio = 'Policia';
  4. jugadors[0].nivell = 1;
  5. jugadors[0].faccio = "Blaus de Cambrils";
  6.  
  7. jugadors[1].nom = 'Patrici';
  8. jugadors[1].cognom = 'Montnegre';
  9. jugadors[1].ocupacio = 'Delinqüent';
  10. jugadors[1].nivell = 2;
  11. jugadors[1].faccio = "Vermells de Reus";

Com que els dos jugadors es creen a partir de jugador, el més pràctic és afegir el mètode toString() a aquest objecte, de manera que serà accessible per tots els jugadors.

La implementació és molt simple, només s’ha de generar una cadena de text amb la informació i una vegada implementada només cal cridar el mètode i mostrar-lo per la consola per demostrar que funciona correctament: 

  1. jugador.toString = function() {
  2.   let dadesFormatades = `${this.nom} ${this.cognom}. ${this.ocupacio} , nivell ${this.nivell} (´ ${this.faccio})`;
  3.   return dadesFormatades;
  4. }
  5.  
  6. console.log(jugadors[0].toString());
  7. console.log(jugadors[1].toString());

Per eliminar una propietat només cal fer servir l’operador delete: 

  1. delete (jugadors[1].faccio);

Tal com es pot comprovar, en eliminar la facció del jugador es mostra el valor de la propietat de l’objecte original: jugador. Per tant, es mostrarà: Sense facció.

Funcionament dels prototipus

L’objectiu d’aquesta activitat és comprovar el funcionament de la propietat prototype i el comportament de la cadena de prototipus.

L’empresa Entreteniments IOC, SL vol ampliar el tipus de zombis del seu pròxim llançament. Compten amb un constructor de zombis per generar un tipus bàsic que els permet canviar algunes propietats bàsiques, però volen implementar diferents monstres amb comportaments diferents.

Us demanen que codifiqueu nous objectes que aprofitin el constructor ja implementat i permetin crear monstres amb les següents característiques:

  • Monstres aquàtics que puguin nadar.
  • Abominacions que puguin atacar diverses vegades.

Aquestes característiques s’han de poder afegir a qualsevol monstre ja generat, de manera que no han de formar part d’un constructor.

Us proporcionen el codi següent, amb una demostració del comportament actual: 

  1. let Zombi = function(nom, puntsVida, potencia) {
  2.   this.nom = nom;
  3.   this.puntsVida = puntsVida;
  4.   this.maxVida = puntsVida;
  5.   this.potencia = potencia;
  6. }
  7.  
  8. Zombi.prototype.atacar = function(objectiu) {
  9.   console.log(this.nom + " ataca a " + objectiu + " i li causa " + this.potencia + " punts de dany");
  10. }
  11.  
  12. Zombi.prototype.moure = function(tipusTerreny) {
  13.   if (tipusTerreny === "aigua") {
  14.     console.log(this.nom + "No es pot moure a través d'aquest tipus de terreny");
  15.   } else {
  16.     console.log(this.nom + " es mou a través del " + tipusTerreny);
  17.   }
  18. }
  19.  
  20. let zombi = new Zombi('Bob', 2, 3);
  21. zombi.atacar('Ricard');
  22. zombi.moure('bosc');

El primer que s’ha de fer és generar les noves funcions amb el comportament indicat: 

  1. let nadar = function(tipusTerreny) {
  2.   if (tipusTerreny === "aigua") {
  3.     console.log(this.nom + " es mou a través del " + tipusTerreny);
  4.   } else {
  5.     console.log(this.nom + " no es pot moure a través d'aquest tipus de terreny");
  6.   }
  7. }
  8.  
  9. let atacMultiple = function(objectiu) {
  10.   let nombreAtacs = 3;
  11.   for (let i = 0; i < nombreAtacs; i++) {
  12.     Zombi.prototype.atacar.apply(this, arguments);
  13.   }
  14. }

En el primer cas es reescriu la funció sense fer cap comprovació, però en el segon cas reaprofita el comportament original invocant el mètode atacar() del prototipus de Zombi diverses vegades. Com que es fa servir el mètode apply() s’ha de passar el context (this) i els arguments.

A continuació s’ha de generar una instància d’un objecte fent servir el constructor zombi, augmentem l’objecte afegint un mètode moure() al qual assignem la funció nadar(). Recordeu que la instància de l’objecte no té cap mètode, s’han afegit al prototipus del constructor: 

  1. let popZombi = new Zombi('Pau, el pop zombi', 15, 3);
  2. popZombi.moure = nadar;
  3. popZombi.moure('aigua');
  4. popZombi.atacar('Ricard');

Com es pot comprovar en el cas de moure() es fa servir el mètode de l’objecte, però a atacar es fa servir el mètode del prototipus de Zombi.

Seguidament s’actualitza el mètode atacar() amb la funció atacMultiple() i se’n comprova el funcionament: 

  1. popZombi.atacar = atacMultiple;
  2. popZombi.atacar('Ricard');

Tal com es pot apreciar, ara l’objecte compta amb dos mètodes propis i, per tant, ja no crida els mètodes del prototipus.

Creació d'objectes

L’objectiu d’aquesta activitat és reforçar els coneixements sobre els patrons de construcció d’objectes i aplicar el més adient als requeriments especificats.

El joc de zombis en el que està treballant Entreteniments IOC, SL està a punt d’entrar en fase beta, és a dir, començarà a ser provat per persones alienes a l’empresa. Per aquesta raó cal encapsular la informació per evitar que usuaris malintencionats puguin modificar les dades.

Us han encarregat que utilitzeu alguna tècnica (patró de generació) per adaptar l’estructura de dades del jugador per evitar que aquestes puguin ser manipulades externament, però ha de ser possible accedir al nom i al mètode toString() i heu de demostrar-ne el funcionament.

A continuació podeu veure l’estructura que us proporcionen per adaptar: 

  1. let jugador = {
  2.   nom: 'Nom del jugador',
  3.   cognom: 'Cognom del jugador',
  4.   ocupacio: 'Sense ocupació',
  5.   nivell: 0,
  6.   faccio: 'Sense facció',
  7.   incrementarNivell: function() {
  8.     nivell++
  9.   },
  10.   toString: function() {
  11.     let dadesFormatades = `${this.nom} ${this.cognom}. ${this.ocupacio}, ${nivell} ${this.nivell} (${this.faccio})`;
  12.     return dadesFormatades;
  13.   }
  14. }

Com que es demana poder encapsular la informació hi ha dues opcions:

  • Aplicar el patró funcional.
  • Aplicar el patró constructor.

Qualsevol de les dues tècniques es pot fer servir per resoldre aquesta activitat. En aquesta solució s’ha optat per fer servir el patró constructor. 

  1. let Jugador = function(_nom, _cognom, _ocupacio) {
  2.   let _faccio = 'Sense facció';
  3.   let _nivell = 0;
  4.   let _incrementarNivell = function() {
  5.     nivell++;
  6.   };
  7.  
  8.   this.getNom = function() {
  9.     return _nom;
  10.   }
  11.   this.toString = function()  {
  12.     let dadesFormatades = `${_nom} ${_cognom}. ${_ocupacio}, ${nivell} ${_nivell} (${_faccio})`;
  13.     return dadesFormatades;
  14.   }
  15. }
  16.  
  17. let ricard = new Jugador('Ricard', 'Enjuts', 'Policia');
  18. console.log(ricard.getNom());
  19. console.log(ricard.toString());

Fixeu-vos que s’ha fet servir el prefix ‘_’ perquè sigui més clar quines variables i paràmetres seran considerats privats: _nom, _cognom, _ocupacio, _faccio, _nivell i el mètode _incrementarNivell().

Per altra banda, tenim dos mètodes públics: getNom() i toString(). Com que es tracta de mètodes públics han d’anar prefixats per this, per fer referència a la mateixa instància de l’objecte.

Gràcies a la clausura que es crea al constructor, tots dos mètodes públics continuen tenint accés a les variables i paràmetres de la funció, encara que l’execució del constructor hagi finalitzat.

Herència (protipus)

L’objectiu d’aquesta activitat és aplicar alguna de les tècniques d’herència conegudes per crear una jerarquia d’objectes mitjançant la herència per prototipus.

Us han proporcionat el següent constructor d’objectes de tipus Consumible. Volen poder generar consumibles de diferents tipus, de manera que cada un podrà tenir efectes completament diferents. Us han proporcionat un objecte per comprovar quin ha de ser el funcionament. El codi és el següent: 

  1. let Consumible = function(_nom, _efecte) {
  2.   this.getNom = function() {
  3.     return _nom;
  4.   }
  5.  
  6.   this.getEfecte = function() {
  7.     return _efecte;
  8.   }
  9. }
  10.  
  11. Consumible.prototype.consumir = function(personatge) {
  12.   console.log(`${personatge.nom} consumeix ${this.getNom()} i ${this.getEfecte()}`);
  13. }
  14.  
  15. let poma = new Consumible('Poma', 'no té cap efecte');
  16. let ricard = {
  17.   nom: 'Ricard',
  18.   potencia: 1,
  19.   puntsVida: 8,
  20.   maxPuntsVida: 10,
  21.   consumir: function(consumible) {
  22.     consumible.consumir(this);
  23.   }
  24. }
  25.  
  26. ricard.consumir(poma);

Podeu veure aquest codi en l’enllaç següent: codepen.io/ioc-daw-m06/pen/ExyQBoX?editors=0012.

  • El primer tipus d’objecte ha de permetre generar objectes que recuperin punts de vida fins la quantitat definida per paràmetre al constructor i respectant el màxim de punts de vida del personatge.
  • El segon tipus d’objecte ha d’augmentar la potència del personatge però, per contra, abaixar el màxim de punts de vida i ajustar els punts de vida actuals a aquest nou màxim si és necessari.

Primerament es crea un nou constructor, anomenat RecuperadorVida(). Dins del constructor s’invoca el constructor Consumible(), passant com a arguments _nom i _efecte. Com que s’ha afegit una nova propietat privada, cal afegir un mètode per accedir-hi, que s’ha anomenat getPotencia().

A continuació cal modificar el prototipus de RecuperadorVida perquè sigui igual al de Consumible. Recordeu que s’ha de crear una còpia amb Object.create() per evitar que qualsevol canvi al prototipus afecti també a Consumible. A més, s’ha de corregir el constructor i guardem una referència al pare (parent). En aquest cas sí que ha de ser una referència al prototipus de Consumible directament, ja que aquesta propietat s’ha de tractar com si fos de només lectura perquè s’utilitzarà exclusivament per accedir als mètodes del constructor pare.

Finalment s’afegeix el nou comportament que comprova si els punts de vida ja es troben al màxim. Si són menors, els augmenta segons la potència i el màxim possible i en cas contrari mostra un missatge. 

  1. let RecuperadorVida = function(_nom, _efecte, _potencia) {
  2.   Consumible.call(this, _nom, _efecte);    
  3.  
  4.   this.getPotencia = function() {
  5.     return _potencia;
  6.   }
  7. }
  8.  
  9. RecuperadorVida.prototype = Object.create(Consumible.prototype);
  10. RecuperadorVida.prototype.constructor = RecuperadorVida;
  11. RecuperadorVida.prototype.parent = Consumible.prototype;
  12.  
  13. RecuperadorVida.prototype.consumir = function(personatge) {
  14.   if (personatge.puntsVida < personatge.maxPuntsVida) {
  15.  
  16.     personatge = Math.min(personatge.puntsVida + this.getPotencia(), personatge.maxPuntsVida)
  17.     RecuperadorVida.prototype.parent.consumir.call(this, personatge)
  18.   } else {
  19.     console.log(`${personatge.puntsVida} ja ha recuperat tota la salut.`);
  20.   }
  21. }
  22.  
  23. let plantaCurativa = new RecuperadorVida('Planta curativa', 'recupera punts de vida', 2);
  24.  
  25. ricard.consumir(plantaCurativa);
  26. ricard.consumir(plantaCurativa);
  27. ricard.consumir(plantaCurativa);

Implementar el següent element de la jerarquia de classes és encara més simple. Com que tant Enfurismador() com RecuperadorVida() tenen la mateixa signatura (es passen els mateixos paràmetres) podem fer que el primer hereti de RecuperadorVida() (tot i que seria més adient canviar el nom del constructor per algun altre de més genèric).

Com que en la implementació del nou mètode consumir() no cal cridar al pare, no cal guardar-ne cap referència i es pot implementar el mètode directament: 

  1. let Enfurismador = function(_nom, _efecte, _potencia) {
  2.   RecuperadorVida.apply(this, arguments);
  3. }
  4.  
  5. Enfurismador.prototype = Object.create(RecuperadorVida.prototype);
  6. Enfurismador.prototype.constructor = Enfurismador;
  7.  
  8. Enfurismador.prototype.consumir = function(personatge) {
  9.   if (personatge.maxPuntsVida > 1) {
  10.     personatge.maxPuntsVida--;
  11.     personatge.puntsVida = Math.min(personatge.puntsVida, personatge.maxPuntsVida);
  12.     personatge.potencia += this.getPotencia();
  13.     console.log(`Grgrgrgr! ${personatge.nom} consumeix ${this.getNom()} i s\´enfurisma (potencia: ${personatge.potencia})`);
  14.   } else {
  15.     console.log('No te cap efecte');
  16.   }
  17. }
  18.  
  19. let banyaSegol = new Enfurismador('Banya del sègol', 's\´enfuria', 1);
  20.  
  21. ricard.consumir(banyaSegol);
  22. ricard.consumir(banyaSegol);
  23. ricard.consumir(banyaSegol);
  24. ricard.consumir(banyaSegol);
  25. ricard.consumir(banyaSegol);
  26. ricard.consumir(banyaSegol);
  27. ricard.consumir(banyaSegol);

Herència (classes)

L’objectiu d’aquesta activitat és crear una jerarquía de classes.

Us han proporcionat les següent classes Consumible i Persona. Volen poder generar consumibles de diferents tipus, de manera que cada un podrà tenir efectes completament diferents. Us han proporcionat un objecte per comprovar quin ha de ser el funcionament. El codi és el següent: 

  1. class Consumible {
  2.   constructor(nom, efecte) {
  3.     this._nom = nom;
  4.     this._efecte = efecte;
  5.   }
  6.  
  7.   get nom() {
  8.     return this._nom;
  9.   }
  10.  
  11.   get efecte() {
  12.     return this._efecte;
  13.   }
  14.  
  15.   consumir(personatge) {
  16.     console.log(`${personatge.nom} consumeix ${this.nom} i ${this.efecte}`);    
  17.   }
  18. }
  19.  
  20. class Persona {
  21.   constructor(nom, potencia, puntsVida, maxPuntsVida) {
  22.     this.nom = nom;
  23.     this.potencia = potencia;
  24.     this.puntsVida = puntsVida;
  25.     this.maxPuntsVida = maxPuntsVida;
  26.   }
  27.  
  28.   consumir(consumible) {
  29.     consumible.consumir(this);
  30.   }
  31. }
  32.  
  33. let poma = new Consumible('Poma', 'no té cap efecte');
  34. let ricard = new Persona('Ricard', 1, 8, 10);
  35.  
  36. ricard.consumir(poma);

Podeu veure aquest codi en l’enllaç següent: codepen.io/ioc-daw-m06/pen/NWryZQx?editors=0012.

  • El primer tipus de classe ha de permetre generar objectes que recuperin punts de vida fins a la quantitat definida per un nou paràmetre que tindrà el constructor, però, a la vegada, respectant el màxim de punts de vida del personatge.
  • El segon tipus de classe ha d’augmentar la potència del personatge però, a la vegada, abaixar el màxim de punts de vida en una unitat i ajustar els punts de vida actuals a aquest nou màxim si és necessari.

Primerament es crea una nova classe, anomenada RecuperadorVida com a subclasse de Consumible. Al constructor se li passen els paràmetres nom, efecte i potencia. Els dos primers es passen al constructor de la classe pare i potencia es desa com a propietat d’aquesta classe.

Seguidament, afegim la funció consumir(), que reemplaçarà a la funció consumir() original. Fixeu-vos que, un cop es realitza la recuperació de punts de vida, es crida al mètode de la classe pare per mostrar el missatge. 

  1. class RecuperadorVida extends Consumible {
  2.   constructor(nom, efecte, potencia) {
  3.     super(nom, efecte);
  4.     this._potencia = potencia;
  5.   }
  6.  
  7.   get potencia() {
  8.     return this._potencia;
  9.   }
  10.  
  11.   consumir(personatge) {
  12.     if (personatge.puntsVida < personatge.maxPuntsVida) {
  13.       personatge.puntsVida = Math.min(
  14.         personatge.puntsVida + this.potencia,
  15.         personatge.maxPuntsVida
  16.       );
  17.       super.consumir(personatge);
  18.     } else {
  19.       console.log(`${personatge.puntsVida} ja ha recuperat tota la salut.`);
  20.     }
  21.   }
  22. }
  23.  
  24. let plantaCurativa = new RecuperadorVida(
  25.   "Planta curativa",
  26.   "recupera punts de vida",
  27.   2
  28. );
  29.  
  30. ricard.consumir(plantaCurativa);
  31. ricard.consumir(plantaCurativa);
  32. ricard.consumir(plantaCurativa);

A continuació implementem la classe Enfurismador. En aquest cas podríem haver-hi fet servir com a classe pare RecuperadorVida, ja que el constructor de totes dues classes inclou la potència, però això podria portar a confusió, ja que si s’invoqués el mètode consumir() del pare (com es fa a RecuperadorVida per mostrar el missatge) s’aplicaria l’efecte restaurador de vida. 

  1. class Enfurismador extends Consumible {
  2.  
  3.   constructor (nom, efecte, potencia) {
  4.     super(nom, efecte);
  5.     this._potencia = potencia;
  6.   }
  7.  
  8.   get potencia() {
  9.     return this._potencia;
  10.   }
  11.
