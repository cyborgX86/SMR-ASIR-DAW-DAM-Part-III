Resum

És important conèixer les diferents funcions que ens proporciona el llenguatge JavaScript. Les funcions més importants per tractar cadenes de text són les següents:

  • lenght: calcula la longitud d’una cadena de text.
  • concat: uneix diverses cadenes de text.
  • toUpperCase: transforma tots els caràcters de la cadena als seus corresponents caràcters en majúscules.
  • toLowerCase: transforma tots els caràcters de la cadena als seus corresponents caràcters en minúscules.
  • charAt: obté el caràcter que es troba en una posició.
  • indexOf: calcula la posició en la qual es troba el caràcter indicat dins de la cadena de text
  • lastIndexOf: calcula l’última posició en la qual es troba el caràcter indicat dins de la cadena de text.
  • substring: extreu una porció d’una cadena de text.
  • split: converteix una cadena de text en un array de cadenes de text.

Les funcions predefinides de nombres que s’expliquen són les següents:

  • NaN: s’utilitza per indicar un valor numèric no definit.
  • isNaN: funció que permet protegir l’aplicació de possibles valors numèrics no definits.
  • Infinity: fa referència a un valor numèric infinit i positiu.
  • toFixed: funció que retorna el nombre original amb punts decimals i realitza els arrodoniments necessaris.

Altres funcions importants de la biblioteca JavaScript són les següents:

  • alert: funció que s’utilitza quan es vol donar una informació a l’usuari sense esperar cap resposta d’ell.
  • confirm: funció que s’utilitza quan es vol donar l’opció a l’usuari de confirmar o no l’execució d’una tasca del programa.
  • prompt: funció que s’utilitza per demanar a l’usuari alguna informació.
  • parseInt: funció que converteix el seu primer argument en un enter.
  • parseFloat: funció que converteix un string a un número de punt flotant sempre que sigui possible.
  • setTimeout: executa una funció després d’un instant de temps.
  • setInterval: executa una funció indefinidament.

Qui programa també pot definir diferents tipus de funcions.

Els àmbits de les variables són una mica diferents dels altres llenguatges de programació. L’àmbit d’una declaració creada dins d’un bloc no acaba (com passa en altres llenguatges) al final del bloc. Un bloc pot ser una estructura de control com per exemple un bucle del tipus for.

L’àmbit d’una variable és la zona del programa en la qual es defineix la variable. JavaScript defineix dos àmbits per a les variables: global i local. En JavaScript, les funcions són les que declaren els àmbits i no els blocs.

Totes les funcions pertanyen a algun objecte, fins i tot les declarades a l’inici del programa. L’objecte ‘pare’ és l’objecte window. Aquest objecte té com a propietat totes les variables declarades com a globals i com a mètodes totes les funcions declarades pel programador. S’ha d’intentar no declarar funcions que no tinguin sentit com a mètode d’aquest objecte. Per aquest motiu, apareix el concepte de funcions sense nom o funcions autoexecutables.

És un llenguatge funcional on les funcions són la base del llenguatge. Una funció es pot invocar fora del seu àmbit de declaració. Per exemple, la funció setInterval permet executar qualsevol funció independentment de si utilitza o no variables globals o locals. Per solucionar aquest problema, JavaScript, ha creat les clausures.

Les clausures són com bombolles de seguretat de les funcions. JavaScript és un llenguatge funcional i les funcions poden ser executades fora del seu context, per això, necessita que les variables a les quals té accés quan es declara existeixin quan s’executa. Aquest és un dels punts clau a l’hora d’entendre aquest llenguatge. Les funcions es poden executar fora del seu context i per tant les variables a les quals hauria de tenir accés podrien no existir. Les clausures evita que aquestes variables no existeixin. Les variables poden viure més enllà del seu context si es preveu que una funció hauria de poder accedir al contingut de la variable.

Per crear un array existeixen dues maneres, utilitzant el constructor o de manera literal. La diferència és que, si s’utilitza el constructor, es pot definir una mida inicial a l’array.

Les principals propietats i funcions que s’expliquen dels arrays són les següents:

  • length: dóna el nombre d’elements que conté.
  • concat: funció que es fa servir per crear un nou array amb els elements de dos arrays diferents.
  • push: funció que afegeix un element al final de l’array.
  • unshift: funció que afegeix els elements a l’inici de l’array.
  • pop: funció que elimina l’últim element de l’array.
  • shift: funció que elimina el primer element de l’array.
  • sort: funció que ordena alfabèticament els elements d’unarray.
  • split: converteix una cadena de text en un array de cadenes de text.
  • join: funció que uneix tots els elements d’un array en una cadena de text utilitzant un caràcter d’unió.
  • reverse: funció que modifica un array col·locant els seus elements en l’ordre invers a la seva posició original.
  • indexOf: retorna la posició de l’array on es troba la primera ocurrència del valor cercat.
  • lastIndexOf: retorna la posició de l’array on es troba l’última ocurrència del valor cercat.
  • every(function test()): funció que comprova, element a element, si tots els valors emmagatzemats a l’array compleixen la funció test.
  • some(function test()): funció que comprova si hi ha algun element que compleix la funció test que es passa a la funció.
  • filter(function test(){}): crea un array i l’omple amb els valors que satisfan la funció test.
  • slice(inici, final): funció que retorna un nou array amb els valors que es troben des de l’índex inici fins a l’índex final.
  • splice: funció que afegeix i elimina elements a partir d’una posició de l’array.

Quan s’utilitzen les funcions anteriors per augmentar la funcionalitat del programa ens veiem amb la necessitat de modificar el codi per tal de no repetir fragments o per reaprofitar les funcions creades. Aquesta reestructuració del codi s’anomena fer refacció. Fer refacció (Code Refactoring) vol dir, en programació, reestructurar el codi per tal de millorar-lo i reduir la seva complexitat sense canviar les funcionalitats que hi havia fins al moment. Aquest procés és indispensable en qualsevol etapa que impliqui millorar o incorporar noves característiques a l’aplicació.

Taula de continguts: Ofereix una vista general de l'estructura del mòdul, que us facilitarà la navegació a través dels seus continguts.
Opcions de format: Permet configurar el format de lectura a partir de les vostres preferències, modificant la mida de la lletra, el color del fons, l'amplada de la pàgina o l'ocultació dels continguts complementaris, entre d'altres.
Imprimir: Envia el contingut seleccionat a la impressora.
Prefereits: Permet desar aquelles parts del contingut a les que us interessi accedir en un moment posterior; us permetrà anar creant un repositori personalitzat de les seccions que considereu més rellevants. Un cop l'hageu començat a fer servir se us mostrarà a sota un comptador que us informarà del nombre de preferits que s'hagin emmagatzemat fins al moment.
Comptador i llistat de preferits: Indica el nombre de preferits que s'han desat fins el moment; clicant damunt seu accedireu al llistat de preferits, i a través d'aquest podreu accedir directament als continguts que hageu emmagatzemat.
Ajuda: Aquesta opció activa la informació de les funcionalitats del web tot situant el punter del ratolí al damunt dels diferents ítems.
Mostrar/Ocultar barra lateral: Deshabilita la barra d'opcions, permetent la seva recuperació quan es desitgi.
Cerca: Introduïu les paraules clau sobre aquells conceptes que desitgeu localitzar en els continguts del mòdul. També podeu excloure un terme de la cerca tot afegint-hi el signe “-” al davant.
Capçalera: Indica l'apartat o secció que es mostra en pantalla. Tingueu present que les capçaleres de segon, tercer i quart nivell també es poden desar com a marcadors.
